# GAS Implementation Rules

## GAS 固有制約

| 制約 | 内容 |
|---|---|
| 実行時間上限 | 6分（1トリガーで処理できる名刺は最大約20枚） |
| ES Modules | 使用不可。`import`/`export` は書かない |
| npm | 使用不可。外部ライブラリ追加不可 |
| スコープ | 全関数をトップレベルに定義（クラス構文も避ける） |

## CONFIG オブジェクト

コードに記載するのは**プレースホルダー文字列のみ**。この形式を崩さない。

```javascript
const CONFIG = {
  // 4フォルダ構成（ファイルライフサイクル管理）
  FOLDER_ID_INPUT:           "ここに01_未処理フォルダのIDを貼り付け",
  FOLDER_ID_OCR_IN_PROGRESS: "ここに02_OCR中フォルダのIDを貼り付け",
  FOLDER_ID_PROCESSED:       "ここに03_処理済みフォルダのIDを貼り付け",
  FOLDER_ID_REVIEW:          "ここに04_要確認フォルダのIDを貼り付け",
  SPREADSHEET_ID:            "ここにスプレッドシートのIDを貼り付け",
  SHEET_NAME:                "シート1"

  // Gemini APIキーはスクリプトプロパティから読み込みます
  // GASエディタ > プロジェクトの設定 > スクリプトプロパティ
  // キー: GEMINI_API_KEY
  // 値: ご自身のAPIキー（決してコードに直書きしないでください）
};
```

## エラーハンドリング

- ループ内の個別ファイル処理は try-catch で囲み、1枚の失敗で全体を止めない
- 外部API呼び出し（`UrlFetchApp.fetch`）は必ず try-catch 済みかレスポンスコードを検証する
- エラーメッセージに実APIキーや内部パスを含めない

```javascript
// GOOD
throw new Error(`Gemini API エラー (Code: ${responseCode})`);

// BAD: レスポンスボディごと渡すとキー漏洩リスク
throw new Error(`Gemini API エラー: ${responseBody}`);
```

## ロギング

GASではロガーが唯一のデバッグ手段のため `console.log` / `console.error` を許可する（本プロジェクト限定）。

- 処理開始・完了・スキップはすべてログに残す
- 件数サマリーを最後に出力する
- ログにAPIキーを含めない

## Gemini API

- モデル: `gemini-1.5-flash`（コスト優先）
- 認証: HTTPヘッダー `x-goog-api-key` を使用（URLパラメータは使用しない）
- `responseMimeType: "application/json"` を必ず指定してJSON強制
- `muteHttpExceptions: true` でHTTPエラーをキャッチ可能にする
- エラーレスポンスはサニタイズしてからログ出力（実装詳細を漏洩させない）
- HEIC/HEIF 画像は `04_要確認` フォルダへ退避（GASでの変換が困難なため）

## ファイル命名・フォルダ構成

4フォルダ構成でファイルのライフサイクルを管理：

| 状態 | フォルダ名 | ファイル名パターン | 説明 |
|---|---|---|---|
| 未処理 | `01_未処理` | （オリジナル名） | アップロード直後 |
| 処理中 | `02_OCR中` | （オリジナル名） | 処理開始時に移動（ロック機構） |
| 処理済み | `03_処理済み` | `[処理済] 会社名_氏名.拡張子` | 正常終了時 |
| 要確認 | `04_要確認` | （オリジナル名） | エラー発生時に退避 |

抽出失敗時のフォールバック: 会社名・氏名とも `"不明"` とする。

## テスト方法

GASにはユニットテストフレームワークがないため、以下の手順で手動確認する。

1. テスト用の名刺画像1枚を `01_未処理` フォルダへアップロード
2. `processBusinessCards()` をスクリプトエディタから手動実行
3. 「実行ログ」でエラーがないことを確認
4. スプレッドシートに行が追加されていることを確認（12列すべて）
5. 「操作ログ」シートに実行履歴が記録されていることを確認
6. 画像が `03_処理済み` に移動し、ファイル名が `[処理済] 会社名_氏名.拡張子` に変わっていることを確認

## スプレッドシートスキーマ

列順は**変更しない**。順序を変える場合は `sheet.appendRow()` の引数も同期して変更する。

### メインシート（名刺台帳）

| 列 | フィールド | 説明 |
|---|---|---|
| A | 登録年月日 | 処理実行日時 |
| B | 氏名 | 抽出された氏名 |
| C | 会社名 | 抽出された会社名 |
| D | 部署・役職 | 部署名と役職 |
| E | 電話番号 | ハイフン付き電話番号 |
| F | メールアドレス | メールアドレス |
| G | 住所 | 住所 |
| H | Webサイト | WebサイトURL |
| I | OCRステータス | 「処理済み」固定（将来拡張用） |
| J | 画像リンク | Google Drive内の画像URL |
| K | 備考 | 手動入力用の空欄 |
| L | OCR生テキスト | Geminiが認識した全テキスト（デバッグ用） |

### 操作ログシート

別シート「操作ログ」に実行履歴を自動記録。

| 列 | フィールド | 説明 |
|---|---|---|
| A | 実行日時 | GAS実行日時 |
| B | ステータス | SUCCESS / ERROR / INFO / TIMEOUT |
| C | メッセージ | 処理内容の説明 |
| D | ファイル数 | 処理したファイル数 |
| E | 詳細 | エラー詳細など |
