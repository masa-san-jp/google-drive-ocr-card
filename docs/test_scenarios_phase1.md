# Phase 1 テストシナリオ（Critical問題修正版）

Phase 1で実装した以下の機能のテストシナリオです。

## 実装済み機能
1. タイムアウト安全装置（4分30秒制限）
2. 4フォルダ構成（01_未処理 → 02_OCR中 → 03_処理済み / 04_要確認）
3. エラー時のファイル退避
4. APIキー最適化（ループ外で1回取得）
5. ヘッダー認証（`x-goog-api-key`）
6. エラーメッセージのサニタイズ

---

## テストケース一覧

### TC-001: 正常系 - 単一ファイルの処理

**目的**: 正常な名刺画像が4フォルダを正しく遷移するか確認

**前提条件**:
- CONFIG に4つのフォルダIDが正しく設定されている
- Gemini APIキーがスクリプトプロパティに設定されている

**テスト手順**:
1. テスト用の名刺画像1枚を `01_未処理` フォルダにアップロード
2. GASエディタから `processBusinessCards()` を手動実行
3. 実行ログを確認

**期待される結果**:
- [ ] ログに「処理開始: [ファイル名]」が出力される
- [ ] ファイルが一時的に `02_OCR中` へ移動
- [ ] スプレッドシートに1行データが追加される（処理日時、会社名、氏名など）
- [ ] ファイルが `03_処理済み` へ移動
- [ ] ファイル名が「[処理済] 会社名_氏名.jpg」に変更される
- [ ] ログに「処理完了: [新ファイル名]」が出力される
- [ ] ログに「計 1 件の名刺画像を処理しました。」が出力される

## トラブルシューティング

### 問題: ファイルが `02_OCR中` に残ったまま

**原因**: 前回の実行が途中で中断された

**対処法**:
1. `02_OCR中` フォルダのファイルを手動で `01_未処理` に戻す
2. 再度 `processBusinessCards()` を実行

### 問題: 「APIキーが設定されていません」エラー

**原因**: スクリプトプロパティの設定ミス

**対処法**:
1. GASエディタの「プロジェクトの設定」（歯車アイコン）を開く
2. 「スクリプト プロパティ」セクションで `GEMINI_API_KEY` が正しく設定されているか確認
3. キー名は大文字で `GEMINI_API_KEY`（アンダースコア含む）であることを確認

### 問題: 「指定されたシート名が見つかりません」エラー

**原因**: CONFIGの `SHEET_NAME` がスプレッドシートのタブ名と一致していない

**対処法**:
1. スプレッドシートのタブ名を確認（通常は「シート1」）
2. CODE.gs の CONFIG.SHEET_NAME と一致させる

### 問題: タイムアウト安全装置が発動しない

**原因**: ファイル数が少ない、または処理が高速

**対処法**:
- テスト用に TIMEOUT_MS を一時的に短くする（例: `30 * 1000` = 30秒）
- 大量のファイル（20枚以上）でテストする
