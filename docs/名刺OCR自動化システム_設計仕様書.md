## 1. システム概要

### 1.1 目的

本システムは、スマートフォンのGoogle Driveアプリで撮影した名刺画像を、Google Apps Script（GAS）による自動処理でOCR解析し、Googleスプレッドシートへ連絡先情報として自動登録することを目的とする。社内に高度なプログラマーが不在であることを前提とし、Google Workspaceの標準機能のみで実現する、保守・運用負荷の低いシステムを目指す。

### 1.2 対象ユーザー

- 名刺を受け取る営業担当者・社員全般
- 名刺情報を管理する管理担当者

### 1.3 システム全体フロー

```
📱 スマートフォンで名刺を撮影
        ↓
📂 Google Drive「01_未処理」フォルダに画像が保存される
        ↓
🤖 Google Apps Script が定期的に自動実行（OCR処理）
        ↓
📋 スプレッドシートに氏名・会社名・連絡先が自動登録
        ↓
📁 画像が「03_処理済み」フォルダに移動（ステータス明確化）
```

## 2. 前提条件と制約

### 2.1 環境前提

| 項目 | 詳細 |
|------|------|
| Google Workspace | Business Standard（全社員がアカウント保有） |
| 画像追加手段 | スマートフォンのGoogle Driveアプリでカメラ撮影してアップロード |
| OCRエンジン | Google Drive 内蔵 OCR（追加費用なし） |
| 自動化ツール | Google Apps Script（GAS）：Google Workspaceに標準付属 |
| プログラマー有無 | 高度なプログラマーは社内不在。GASの基本設定ができる担当者が1名いれば運用可能 |
| 対応言語 | 主に日本語名刺（英語名刺も一定程度対応可能） |

### 2.2 方針

- 外部サービス・有料APIは使用しない（Google Workspace標準機能のみ）
- 担当者が日常的に操作しやすいシンプルな構成にする
- OCR処理のステータスを視覚的に明確に区別できるようにする
- OCRの精度限界を前提とし、必ず人手による確認・修正ステップを設ける

## 3. Google Drive フォルダ構成

### 3.1 フォルダ構造

以下のフォルダを「名刺管理」という親フォルダ内に作成する。

```
名刺管理/
├── 01_未処理/        ← スマホから名刺画像をアップロードするフォルダ。GASが監視する
├── 02_OCR中/         ← GASがOCR処理を開始した際に一時的に画像を移動するフォルダ
├── 03_処理済み/      ← OCR完了後に画像を移動するフォルダ（正常終了）
├── 04_要確認/        ← OCRの信頼度が低く、手動確認が必要と判定された画像を移動するフォルダ
└── 名刺台帳（スプレッドシート）
```

### 3.2 フォルダによるステータス管理

画像がどのフォルダに存在するかで、処理状態を一目で把握できる。

| フォルダ | ステータス | 次のアクション |
|----------|------------|----------------|
| `01_未処理/` | 未処理（GAS待ち） | GASが自動で処理を開始 |
| `02_OCR中/` | OCR処理中 | 処理完了を待つ |
| `03_処理済み/` | OCR完了・スプレッドシート登録済み | スプレッドシートで内容確認・修正 |
| `04_要確認/` | 要手動確認 | 画像を目視して手動でスプレッドシートに入力 |

## 4. スプレッドシート（名刺台帳）仕様

### 4.1 シート構成

スプレッドシートは以下の2つのシートで構成する。

- **名刺台帳シート**：OCR抽出データと管理情報を格納するメインシート
- **操作ログシート**：GASの実行履歴・エラー記録を格納するシート

### 4.2 名刺台帳シート　列定義

| 列 | 列名 | 型 | 説明 |
|----|------|----|------|
| A | 登録年月日 | 日付 | GASがスプレッドシートに書き込んだ年月日（自動） |
| B | 氏名 | テキスト | OCR抽出した氏名 |
| C | 会社名 | テキスト | OCR抽出した会社・組織名 |
| D | 部署名・役職 | テキスト | OCR抽出した部署名や役職・肩書き |
| E | 電話番号 | テキスト | OCR抽出した電話番号（モバイル端末があれば優先して記載） |
| F | メールアドレス | テキスト | OCR抽出したメールアドレス |
| G | 住所 | テキスト | OCR抽出した住所 |
| H | WebサイトURL | テキスト | OCR抽出したWebサイトURL |
| I | OCRステータス | テキスト | 「処理済み」「要確認」のいずれか。セル色で視覚化（後述） |
| J | 画像リンクURL | URL | 処理済みフォルダ内の画像へのリンク（確認用） |
| K | 備考 | テキスト | 担当者が自由に入力できるメモ欄 |
| L | OCR生テキスト | テキスト | GASがOCRで取得した全テキスト生データ（確認・修正用参考情報） |

### 4.3 OCRステータスの色分け仕様

I列「OCRステータス」のセルは、GASが以下の条件付き書式を自動適用し、視覚的に識別できるようにする。

| ステータス値 | セル色 | 意味 |
|--------------|--------|------|
| 処理済み | 🟢 緑 | OCR正常完了。スプレッドシートに登録済み |
| 要確認 | 🔴 赤 | OCRの信頼度が低い。担当者が画像を参照して手動修正が必要 |
| 処理中 | 🔵 青 | GASが現在処理中（通常は数分で完了） |

## 5. Google Apps Script（GAS）仕様

### 5.1 スクリプト構成

| 関数名 | 役割 |
|--------|------|
| `main()` | トリガーから呼ばれるメイン関数。未処理フォルダを監視し処理を順次実行する |
| `ocrImage(fileId)` | Google Drive APIを使って画像をGoogle ドキュメントに変換（OCR実行）し、テキストを抽出する |
| `parseBusinessCard(text)` | OCRテキストから氏名・会社名・電話番号・メール等を正規表現でパースして構造化データを返す |
| `writeToSheet(data)` | 名刺台帳スプレッドシートに新しい行として抽出データを書き込む |
| `moveFile(fileId, toFolderId)` | 画像ファイルを指定フォルダに移動する（フォルダ間のステータス変更） |
| `writeLog(msg, type)` | 操作ログシートに実行日時・処理内容・エラー内容を記録する |

### 5.2 トリガー設定

GASの「トリガー」機能を使い、定期的に自動実行する。外部プログラムや常時起動サーバーは不要。

| 設定項目 | 推奨値 |
|----------|--------|
| 実行関数 | `main` |
| トリガーの種類 | 時間主導型 |
| 時間の間隔 | 1時間ごと（業務時間帯：8時〜20時に設定推奨） |
| エラー通知 | GASのデフォルト機能でエラー時にメール通知（管理者のGmailアドレス宛） |

> **補足：リアルタイム処理について**
> Google Driveにはフォルダへのファイル追加をリアルタイムで検知するトリガー機能が現時点では提供されていない。そのため、本システムは定期実行（最短15分間隔）による擬似リアルタイム処理を採用する。緊急性が高い場合は管理者が手動で `main()` を実行することも可能。

### 5.3 OCR処理の詳細ロジック

#### 5.3.1 OCR実行方法

Google Drive APIには、画像ファイルをGoogle ドキュメント形式に変換する際にOCRを自動実行する機能がある。GASからこの機能を呼び出すことで、追加費用なしにOCRを実現する。

| 順 | 処理内容 |
|----|----------|
| 1 | `01_未処理` フォルダ内の画像ファイル（jpg, png, heic等）を一覧取得する |
| 2 | 画像ファイルを `02_OCR中` フォルダに移動し、スプレッドシートに「処理中」ステータスで行を追加する |
| 3 | `Drive.Files.copy` APIで画像をGoogle ドキュメント形式（`mimeType: application/vnd.google-apps.document`）にコピー変換する。このコピー時にOCRが実行される |
| 4 | 生成されたGoogle ドキュメントからテキストを取得し、`parseBusinessCard()` 関数で構造化する |
| 5 | 抽出データをスプレッドシートに書き込む。OCR生テキストの文字数が一定以上あれば「処理済み」、少ない場合は「要確認」とする |
| 6 | 画像ファイルを `03_処理済み` または `04_要確認` フォルダに移動する。一時的に生成したGoogle ドキュメントは削除する |
| 7 | 処理結果を操作ログシートに記録する |

#### 5.3.2 テキストパース（構造化）ロジック

OCRで取得したテキスト全体から、以下のルールで各フィールドを抽出する。完全な自動化は困難なため、抽出できなかった項目はすべて「要確認」として担当者に通知する。

| フィールド | 抽出方法 | 例・補足 |
|------------|----------|----------|
| メールアドレス | 正規表現（`@` を含む文字列） | `example@company.co.jp` |
| 電話番号 | 正規表現（数字とハイフン） | `03-1234-5678`、`090-XXXX-XXXX` 等 |
| WebサイトURL | 正規表現（`http`/`https`/`www`） | `https://www.company.co.jp` |
| 住所 | 都道府県名・市区町村名のキーワードマッチ | 「東京都」「大阪府」等で始まる行 |
| 会社名 | 「株式会社」「（株）」「有限会社」等のキーワードを含む行 | `株式会社○○`、`○○ Co.,Ltd.` 等 |
| 役職 | 「部長」「課長」「代表」「Director」等のキーワードマッチ | 営業部長、マネージャー等 |
| 氏名 | 上記フィールドに合致しない最も短い行（通常2〜4文字）をヒューリスティックで推定 | 精度が最も低いため「要確認」判定の主要因となる |

## 6. エラーハンドリングと通知

### 6.1 エラー種別と対応

| エラー種別 | 発生条件 | 対応方法 |
|------------|----------|----------|
| OCRテキスト不足 | 取得テキストが20文字未満 | ステータスを「要確認」にして `04_要確認` フォルダへ移動。担当者に手動入力を促す |
| 権限エラー | フォルダへのアクセス権がない | ログに記録し、管理者にメール通知する。画像は移動せず元フォルダに残す |
| 処理タイムアウト | GAS実行時間が6分を超過 | GASの最大実行時間（6分）を超えると自動停止。次回トリガー時に未処理のファイルから処理を再開する |
| 非対応ファイル形式 | PDFや動画など名刺以外のファイル | `04_要確認` フォルダへ移動し、ログに記録する |

### 6.2 通知設計

- GASのエラー通知機能を有効化し、スクリプトがエラーで停止した場合は管理者のGmailに自動通知される
- 「要確認」のデータが蓄積された場合に週次でリマインドメールを送信するオプション機能を追加することも可能（将来拡張）

## 7. 初期セットアップ手順

本システムのセットアップは、担当者が以下の手順を一度実施するだけでよい。所要時間は約60〜90分を想定する。

| 順 | 作業名 | 詳細 |
|----|--------|------|
| 1 | Google Driveフォルダ作成 | 「名刺管理」フォルダと4つのサブフォルダ（01〜04）を作成する |
| 2 | スプレッドシート作成 | 「名刺台帳」スプレッドシートを作成し、ヘッダー行と条件付き書式を設定する |
| 3 | GASプロジェクト作成 | スプレッドシートのメニュー「拡張機能 > Apps Script」からGASエディタを開き、スクリプトを貼り付ける |
| 4 | フォルダIDの設定 | 各フォルダのURLからフォルダIDをコピーし、GASスクリプトの定数として設定する |
| 5 | Drive APIの有効化 | GASのサービスタブから「Drive API v3」を追加する（OCR機能の利用に必要） |
| 6 | 権限の承認 | GASを初回実行時に、GoogleアカウントへのアクセスをGASに許可する（Googleアカウントでの認証が必要） |
| 7 | トリガーの設定 | GASの「トリガー」メニューから `main()` 関数を1時間ごとに実行するよう設定する |
| 8 | 動作テスト | `01_未処理` フォルダにテスト用名刺画像を1枚アップロードし、GASを手動実行して正常に動作することを確認する |
| 9 | スマートフォン設定案内 | 利用する社員全員に、スマートフォンのGoogle Driveアプリで `01_未処理` フォルダを「スターつき」またはショートカット登録する方法を案内する |

## 8. 日常運用ガイドライン

### 8.1 社員の操作手順（名刺撮影）

1. スマートフォンでGoogle Driveアプリを開く
2. 「名刺管理 > 01_未処理」フォルダに移動する
3. 右下の「＋」ボタンをタップし「カメラを使用」を選択して名刺を撮影する
4. 撮影した画像が自動的にフォルダにアップロードされる
5. 1〜2時間後に名刺台帳スプレッドシートに情報が登録されているので、I列「OCRステータス」を確認する
6. 「要確認」の行は、J列の画像リンクURLから元の名刺画像を参照し、手動で修正・補記する

### 8.2 管理者の定期確認（月次推奨）

- 操作ログシートにエラーが蓄積していないか確認する
- `04_要確認` フォルダに長期間残っている画像がないか確認し、担当者に入力を促す
- スプレッドシートの重複データ（同一人物の名刺を複数回登録）を整理する
- GASトリガーが正常に動作しているか確認する

## 9. 制限事項と注意事項

> ⚠️ **OCR精度に関する重要な注意事項**
>
> Google DriveのOCR機能は汎用的なものであり、名刺専用ではない。以下の場合に認識精度が低下する可能性がある。
> - 手書き文字が含まれる名刺
> - デザイン性の高いフォントや背景画像が多い名刺
> - 撮影時のピントが合っていない・光量不足の画像
> - 横書きと縦書きが混在する名刺
>
> **必ずスプレッドシートの内容を目視で確認し、必要に応じて修正してください。**

### 9.1 その他の制限事項

- GASの最大実行時間は6分間のため、1回のトリガーで処理できる画像数は約10〜20枚が目安
- HEICフォーマット（iPhoneのデフォルト形式）はGAS経由のOCRが動作しない場合がある。iPhoneの設定で「カメラ > フォーマット > 互換性優先」（JPEGで保存）に変更することを推奨する
- Google Drive無料版では利用できない機能（Drive API）が含まれるため、Business Standard以上のGoogle Workspaceアカウントが必須
- 個人情報（名刺情報）を扱うため、スプレッドシートと各フォルダのアクセス権限は「リンクを知っている全員が閲覧可能」ではなく、「特定のユーザーのみ」に設定すること

## 10. 将来的な拡張オプション

本システムは最小構成での実装を優先しているが、将来的に以下の拡張を検討できる。

| 拡張オプション | 概要 |
|----------------|------|
| 重複チェック機能 | 同一メールアドレスや電話番号の名刺が既に登録済みの場合、担当者にアラートを表示する |
| Googleコンタクトへの自動連携 | 確認済みの名刺データをGoogleコンタクトに自動でエクスポートし、GmailやGoogleカレンダーと連携させる |
| 週次リマインド通知 | 未確認（要確認）のデータが残っている場合に、担当者にGmailでリマインドを自動送信する |
| 名刺画像の自動リネーム | OCR後に画像ファイル名を「会社名_氏名_日付.jpg」形式に自動リネームする |
| Google Chat通知 | OCR完了時やエラー発生時にGoogle Chatのチャンネルにリアルタイム通知を送る |
