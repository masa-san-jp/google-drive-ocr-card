# 名刺OCRシステム セットアップガイド

このドキュメントは、名刺OCRシステムをGoogle Workspace上で動かすための設定手順です。
プログラミングの知識は不要です。手順に沿って設定をすすめてください。

---

## ステップ1：必要なフォルダとファイルの作成

1. **Google Drive** を開きます。（社内の共有ドライブなど、運用しやすい場所で構いません）
2. 以下の**4つのフォルダ**を新規作成します。
   - `01_未処理` （名刺画像をアップロードする場所）
   - `02_OCR中` （処理中のファイルが一時的に置かれる場所）
   - `03_処理済み` （処理が完了した画像が格納される場所）
   - `04_要確認` （エラーが発生したファイルが退避される場所）
3. 情報を蓄積するための **Google スプレッドシート** を新規作成します。
   - ファイル名は何でも構いません（例：「名刺台帳」など）。
   - タブ名（画面左下）が「シート1」になっていることを確認します。

## ステップ2：各種ID（URL）の取得
設定プログラムに必要なIDを取得します。メモ帳などに控えてください。

1. **01_未処理フォルダのID**
   - 作成した `01_未処理` フォルダをブラウザで開きます。
   - 上部のURLが `https://drive.google.com/drive/folders/〇〇〇〇` となっています。
   - この `〇〇〇〇` （英数字の長い文字列）がフォルダIDです。コピーして控えます。
2. **02_OCR中フォルダのID**
   - 同じように `02_OCR中` フォルダを開き、URLからIDをコピーして控えます。
3. **03_処理済みフォルダのID**
   - 同じように `03_処理済み` フォルダを開き、URLからIDをコピーして控えます。
4. **04_要確認フォルダのID**
   - 同じように `04_要確認` フォルダを開き、URLからIDをコピーして控えます。
5. **スプレッドシートのID**
   - 作成したスプレッドシートを開きます。
   - 上部のURLが `https://docs.google.com/spreadsheets/d/〇〇〇〇/edit` となっています。
   - この `〇〇〇〇` の部分をコピーして控えます。

## ステップ3：Gemini APIキーの取得（無料）
AIに画像解析をさせるための鍵（APIキー）を取得します。

1. [Google AI Studio (https://aistudio.google.com/)](https://aistudio.google.com/) にアクセスし、Googleアカウントでログインします。
2. 画面左側の **「Get API key」** または **「Create API key」** をクリックします。
3. **「Create API key in new project」** をクリックして作成し、表示された長い文字列（APIキー）をコピーして控えます。
   *(※「APIキー」は他人に教えたり公開したりしないでください)*

## ステップ4：Google Apps Script (GAS) の設定

1. 先ほど作成した**スプレッドシート**を開きます。
2. 上部メニューの **「拡張機能」** ＞ **「Apps Script」** をクリックします。
3. 新しい画面（エディタ画面）が開きます。
   - 画面中央に `function myFunction() { ... }` のようなコードが書かれている場合は、**すべて削除**します。
4. 本プロジェクト内にある `src/Code.gs` ファイルの中身を **すべてコピー** して、エディタ画面に貼り付けます。
6. ステップ2で控えた **各種ID** を、「" "」の内側に貼り付けます。
   *(※「GEMINI_API_KEY」はコード内に貼り付けません)*

```javascript
const CONFIG = {
  FOLDER_ID_INPUT: "ここに01_未処理フォルダのIDを貼り付け",
  FOLDER_ID_OCR_IN_PROGRESS: "ここに02_OCR中フォルダのIDを貼り付け",
  FOLDER_ID_PROCESSED: "ここに03_処理済みフォルダのIDを貼り付け",
  FOLDER_ID_REVIEW: "ここに04_要確認フォルダのIDを貼り付け",
  SPREADSHEET_ID: "ここにスプレッドシートのIDを貼り付け",
  SHEET_NAME: "シート1"
};
```

7. 画面上部の **「保存」** ボタン（フロッピーディスクのアイコン）をクリックします。

## ステップ5：Gemini APIキーの安全な登録（スクリプトプロパティ）
APIキーが他人に漏れないよう、安全な領域に登録します。

1. エディタ画面の左端にあるメニュー（歯車アイコンの「**プロジェクトの設定**」を一番下までスクロールします。
2. 「**スクリプト プロパティ**」という項目があるので、「**スクリプト プロパティを追加**」をクリックします。
3. 以下の通りに入力します：
   - プロパティ: `GEMINI_API_KEY` （※すべて大文字、アンダースコアで入力）
   - 値: ステップ3で取得した **APIキーの文字列をそのまま貼り付け**
4. 「**スクリプト プロパティを保存**」ボタンをクリックします。

## ステップ6：プログラムの初回実行と権限の許可
初回実行時のみ、Googleへのアクセス許可（認証）が必要です。

1. 再度、左側のメニューから「**エディタ（< >のアイコン）**」に戻ります。
2. 画面上部のツールバーで**「processBusinessCards」**が選択されていることを確認して、**「実行」** をクリックします。
2. 画面に「**承認が必要です**」と表示されるので、**「権限を確認」** をクリックします。
3. 自分のGoogleアカウントを選択します。
4. 「このアプリは Google で確認されていません」と警告が出る場合があります。（自分が作成したプログラムなので安全です）
   - 左下の **「詳細」** をクリックし、**「無題のプロジェクト（安全ではないページ）に移動」** をクリックします。
5. アクセス権限の画面が表示されるので、一番下の **「許可」** をクリックします。
6. プログラムが実行されます。（実行ログに文字が出力されれば成功です。まだ画像が無いため「計0件の名刺画像を処理しました」と表示されます）

## ステップ7：自動実行（トリガー）の設定
フォルダ内を定期的に監視して自動で処理するように設定します。

1. エディタ画面の左側のメニューアイコンから **「トリガー」**（時計のマーク）をクリックします。
2. 画面右下の **「トリガーを追加」** をクリックします。
3. 以下の通りに設定します：
   - 実行する関数を選択: **processBusinessCards**
   - 実行するデプロイを選択: **Head**
   - イベントのソースを選択: **時間主導型**
   - 時間ベースのトリガーのタイプを選択: **分ベースのタイマー**
   - 時間の間隔を選択（分）: **10分おき** またはお好みの間隔
4. **「保存」** をクリックします。

---

🎉 **以上で設定はすべて完了です！** 🎉

## 使い方・テスト
1. スマートフォンの Google Drive アプリを開きます。
2. `01_未処理` フォルダを開き、「＋」ボタンから「カメラ」を起動し、名刺を撮影して画像を保存します。
3. 最大10分待機すると（またはGASエディタから手動で「実行」を押すと）、以下の処理が自動で行われます：
   - 画像が一時的に `02_OCR中` フォルダへ移動
   - Gemini AIが名刺画像を解析
   - スプレッドシートに名刺情報が自動登録
   - 処理成功時: 画像が `03_処理済み` に移動（ファイル名も「[処理済] 会社名_氏名」に変更）
   - エラー発生時: 画像が `04_要確認` に退避（手動確認が必要）

## フォルダの役割
- **01_未処理**: 名刺画像をアップロードする場所
- **02_OCR中**: 処理中の一時保管場所（通常は空）
- **03_処理済み**: 正常に処理された名刺画像の保管場所
- **04_要確認**: エラーで処理できなかった画像の退避場所（画像が不鮮明、破損などの場合）
