## システム概要

### アーキテクチャ

```
[スマートフォン]
    ↓ 名刺撮影・アップロード
[Google Drive: 01_未処理]
    ↓ GAS定期実行（10分おき）
[Google Apps Script]
    ├─ 02_OCR中（処理中ロック）
    ├─ Gemini API（画像解析）
    └─ スプレッドシート書き込み
    ↓
[Google Drive: 03_処理済み] ← 成功
[Google Drive: 04_要確認]   ← エラー
```

### 技術スタック

| 層 | 技術 | バージョン |
|----|------|----------|
| Runtime | Google Apps Script | V8 |
| AI/OCR | Gemini API | gemini-1.5-flash |
| Storage | Google Drive + Sheets | - |
| 認証 | HTTPヘッダー認証 | x-goog-api-key |

## Phase 1: Critical修正（P0優先度）

### 実装項目

#### 1. APIキーのヘッダー認証への変更
**問題**: URLパラメータでAPIキー送信（ログ漏洩リスク）
**解決策**: HTTPヘッダー `x-goog-api-key` で送信

```javascript
// 変更前
const url = `https://...?key=${apiKey}`;

// 変更後
const url = 'https://...';
const options = {
  headers: { "x-goog-api-key": apiKey }
};
```

**効果**: APIキーがURLログに残らない

#### 2. エラーメッセージのサニタイズ
**問題**: Gemini APIの生エラーレスポンスをログ出力
**解決策**: ステータスコード別の一般化メッセージに置換

```javascript
const errorMessages = {
  400: "無効なリクエスト形式です",
  401: "認証に失敗しました（APIキーを確認してください）",
  429: "API呼び出し制限に達しました",
  500: "Gemini API側でエラーが発生しています"
};
```

**効果**: 実装詳細の漏洩を防止

#### 3. タイムアウト安全装置
**問題**: 大量ファイル処理時にGASの6分制限で強制終了
**解決策**: 4分30秒でループを安全に中断

```javascript
const startTime = new Date().getTime();
const TIMEOUT_MS = 4.5 * 60 * 1000; // 4分30秒

if (elapsedMs > TIMEOUT_MS) {
  console.log("タイムアウト安全装置発動...");
  break;
}
```

**効果**: 処理中ファイルの破損を防止、残りは次回実行で処理

#### 4. 4フォルダ構成によるライフサイクル管理
**問題**: 2フォルダ構成で重複処理・エラー退避ができない
**解決策**: 4フォルダでファイル状態を明確に管理

| フォルダ | 役割 |
|---------|------|
| 01_未処理 | アップロード直後 |
| 02_OCR中 | 処理中（ロック機構） |
| 03_処理済み | 正常終了 |
| 04_要確認 | エラー退避 |

**効果**:
- 重複処理の防止
- エラーファイルの自動退避
- 処理状態の可視化

#### 5. APIキー取得の最適化
**問題**: ループ内で毎回PropertiesServiceアクセス
**解決策**: ループ外で1回のみ取得

**効果**:
- PropertiesServiceアクセス: 10回 → 1回
- APIキー未設定エラーをループ前に検出

#### 6. 重複処理防止機構
**問題**: 並列トリガー実行時の二重処理
**解決策**: 02_OCR中フォルダの存在チェック

```javascript
if (currentParentId === CONFIG.FOLDER_ID_OCR_IN_PROGRESS) {
  console.log("既に02_OCR中フォルダにあります。スキップ");
  continue;
}
```

**効果**: 同一ファイルの重複処理を防止

## Phase 2: 機能拡張（P1-P2優先度）

### 1. スプレッドシート12列スキーマへの拡張

**変更前（8列）**:
```
処理日時 | 会社名 | 部署・役職 | 氏名 | メールアドレス | 電話番号 | 住所 | 画像URL
```

**変更後（12列）**:
```
A: 登録年月日
B: 氏名
C: 会社名
D: 部署・役職
E: 電話番号
F: メールアドレス
G: 住所
H: Webサイト          ← 新規
I: OCRステータス      ← 新規
J: 画像リンク
K: 備考              ← 新規
L: OCR生テキスト      ← 新規
```

**追加フィールドの用途**:
- **Webサイト**: 名刺に記載されたURLを自動抽出
- **OCRステータス**: 「処理済み」を自動記録
- **備考**: 手動入力用の空欄
- **OCR生テキスト**: Geminiが認識した全テキスト（デバッグ用）

### 2. 操作ログシート機能

**新規実装**: `writeLog(logSheet, message, status, fileCount, details)`

**ログシートスキーマ**:
```
実行日時 | ステータス | メッセージ | ファイル数 | 詳細
```

**ログ記録タイミング**:
- タイムアウト発動時（TIMEOUT）
- エラー発生時（ERROR）
- ファイル移動失敗時（ERROR）
- 処理完了時（SUCCESS）

**効果**:
- 実行履歴の自動記録
- エラー追跡の容易化
- システム稼働状況の可視化

### 3. Gemini APIプロンプトの拡張

**追加フィールド**:
```json
{
  "website": "WebサイトURL（http://またはhttps://で始まる）",
  "raw_text": "名刺から認識したすべてのテキスト（改行は半角スペースに置き換え）"
}
```

**効果**: より詳細な名刺データの抽出

## Phase 3: 仕上げと最適化（P2-P3優先度）

### 1. HEIC/HEIF画像の適切な処理

**問題**: HEICをJPEGとして偽装送信（非推奨）
**解決策**: 検出して要確認フォルダへ退避

```javascript
if (mimeType === 'image/heic' || mimeType === 'image/heif') {
  const heicMsg = `HEIC/HEIF形式はサポート対象外です: ${file.getName()}`;
  writeLog(logSheet, heicMsg, "INFO", processedCount,
    "スマートフォンのカメラ設定を「互換性優先（JPEG）」に変更してください");
  file.moveTo(reviewFolder);
  continue;
}
```

**効果**:
- 適切なエラーハンドリング
- ユーザーへのガイダンス提供
- セキュリティリスクの排除
